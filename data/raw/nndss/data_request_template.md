# NNDSS Data Request — Template

**To:** NNDSS.datarequests@health.gov.au
**Subject:** Data request — aggregate imported-case counts for returned-traveller diagnostic tool

---

Dear NNDSS Data Team,

I am writing to request aggregate notification data from the National Notifiable Diseases Surveillance System to support the development of an open-source diagnostic decision support tool for febrile returned travellers in Australian emergency departments.

## Data requested

**Aggregate annual notification counts** (not individual-level data) for the following conditions, stratified by:
- **Place of acquisition:** imported (overseas-acquired) vs locally acquired
- **Year:** 2010-2024 (or most recent available)

### Conditions requested

1. Malaria (with species breakdown if available: P. falciparum, P. vivax, P. ovale, P. malariae)
2. Dengue virus infection
3. Chikungunya virus infection
4. Zika virus infection
5. Typhoid fever
6. Paratyphoid fever
7. Hepatitis A
8. Hepatitis B (acute)
9. Hepatitis E
10. Measles
11. Japanese encephalitis
12. Leptospirosis
13. Q fever
14. Melioidosis
15. Tuberculosis
16. Brucellosis
17. Rickettsial infections (if notifiable at national level)
18. Cholera

## Purpose

These data will be used to construct Australian-weighted diagnostic priors for a Naive Bayes fever differential tool. GeoSentinel surveillance data (which provides destination-diagnosis conditionals) is US/European-weighted; NNDSS imported-case counts will be used to reweight priors to reflect Australia's destination mix (predominantly SE Asia and Pacific).

## Outputs

- The tool and all code will be released open-source (MIT license)
- NNDSS will be cited as the data source
- Only aggregate counts will be used; no individual-level data is required
- A pre-print will be posted on medRxiv, with target submission to Journal of Travel Medicine

## Ethics

This study uses only aggregate published/public data. An HREC exemption determination has been documented under National Statement Section 5.1.22.
